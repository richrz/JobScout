# Archive Index

This folder holds historical and non-binding project records.

## Archive Rules

- Archived docs are preserved for context, not for current product truth.
- Nothing in this folder outranks:
  - `docs/README.md`
  - `docs/decisions/`
  - `docs/product/`
  - `JOURNAL.md`
- If an archived doc still contains useful ideas, those ideas should be promoted into an active spec or ADR rather than linked as current guidance.

## Current Archive Lanes

- `design-history/`
  - superseded visual overhaul and UI revamp planning
- `execution-history/`
  - point-in-time implementation plans, remediation plans, and reports
- `product-direction/`
  - earlier direction-setting docs no longer used as active strategy
- `product-memory/`
  - older temporary memory/bootstrap artifacts
- `session-history/`
  - handoff notes and session-specific pickup context
- `tdd-in-a-box/`
  - archived bundle copy retained for reference

## Still-Useful Historical Records

- `audits/`
  - verification history from earlier task cycles
- `bugs/`
  - targeted bug writeups that may still matter operationally

These records are historical by nature, but they can still be useful references.
